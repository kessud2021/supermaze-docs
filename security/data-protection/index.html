    Data Protection¶

    How SuperMaze protects user data.

    Data Isolation¶

    Client Data¶

    All user data stays local:

    • Username
    • Auth tokens
    • XP and level
    • Leaderboard
    • Settings
    • Theme preferences

    No external servers access this data unless multiplayer is enabled.

    Server Data¶

    Server only stores:

    • Public leaderboard (name + time)
    • Active player positions (temporary)
    • Game settings (if shared)

    No sensitive data on server.

    Data Persistence¶

    localStorage Storage¶

    • Survives page refreshes
    • Survives browser restarts
    • Cleared only by user action
    • No expiration time

    Server Storage¶

    • Player data stored in memory
    • Leaderboard stored in JSON files
    • No database used
    • Data persists across restarts

    Data Protection Measures¶

    XSS Prevention¶

    • All user input sanitized
    • HTML entities encoded
    • Safe DOM manipulation
    • No eval() used

    Input Validation¶

    • Usernames limited to 32 chars
    • URLs validated before use
    • Colors restricted to hex format
    • JSON safely parsed with error handling

    Safe Communication¶

    • WebSocket validates messages
    • Invalid messages rejected
    • No code execution allowed
    • Type checking on all inputs

    Privacy¶

    What Stays Local¶

    • Player passwords (not used)
    • Personal settings
    • Game statistics
    • Play history

    What Can Be Shared¶

    • Username
    • Best times
    • Leaderboard rankings
    • Public statistics

    Clearing Data¶

    Reset Everything¶

    localStorage.clear();
location.reload();

    Clear Specific Data¶

    localStorage.removeItem('maze_leaderboard');
localStorage.removeItem('maze_xp');
localStorage.removeItem('maze_username');

    Your data is your own and stays local unless you enable multiplayer.
