    Code Style¶

    SuperMaze code style guidelines.

    JavaScript¶

    Naming Conventions¶

    • Variables: camelCase
    • Functions: camelCase
    • Constants: UPPER_SNAKE_CASE
    • Classes: PascalCase

    Examples¶

    // Variables
const playerPosition = { x: 0, y: 0 };
let currentScore = 0;

// Functions
function generateMaze() { }
const calculateDistance = () => { };

// Constants
const MAX_PLAYERS = 50;
const GRID_SIZE = 21;

// Objects/Classes
class MazeGenerator { }
const config = { };

    File Organization¶

    • One logical component per file
    • Clear file naming
    • Group related functions
    • Export clear interfaces

    Comments¶

    • Comment complex algorithms
    • Explain "why", not "what"
    • Keep comments updated
    • Use /* */ for blocks, // for lines

    Examples¶

    // Good: Explains the purpose
// Recursive backtracking generates perfect mazes
function generateMaze() { }

// Avoid: States the obvious
// This function generates a maze
function generateMaze() { }

    Function Size¶

    • Keep functions focused
    • Max ~50 lines recommended
    • Single responsibility
    • Easy to test

    Variable Scope¶

    • Use const by default
    • Use let when needed
    • Avoid var
    • Minimize scope

    Indentation¶

    • 2 spaces (not tabs)
    • Consistent throughout
    • Match existing code

    HTML/CSS¶

    HTML¶

    • Semantic elements
    • Proper nesting
    • Meaningful IDs/classes
    • Accessibility attributes

    CSS¶

    • kebab-case for class names
    • Related properties grouped
    • Comments for sections
    • Avoid inline styles

    Examples¶

    <!-- Good -->
<button id="play-btn" class="btn btn-primary">Play</button>

<!-- Avoid -->
<button id="playBtn" class="BTN BLUE">Play</button>

    Error Handling¶

    • Try-catch for risky operations
    • Meaningful error messages
    • Log errors to console
    • Handle gracefully

    Examples¶

    try {
  const data = JSON.parse(jsonString);
  return data;
} catch (error) {
  console.error('Failed to parse JSON:', error);
  return null;
}

    Security¶

    • Always sanitize user input
    • Validate all data
    • Never use eval()
    • Use proper encoding

    Documentation¶

    • Document complex functions
    • Include parameter descriptions
    • Note return types
    • Add examples

 * Calculate distance between two points

 * @param {Object} p2 - Second point {x, y}
 * @returns {number} Distance between points
 */

  // ...
