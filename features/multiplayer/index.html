    Multiplayer¶

    Play SuperMaze with real-time player tracking and coordination.

    Multiplayer Overview¶

    SuperMaze multiplayer allows:

    • Real-time player position tracking
    • WebSocket-based communication
    • Server-coordinated game state
    • Admin dashboard for monitoring
    • Multiple players simultaneously

    Setting Up Multiplayer¶

    Server Requirements¶

    • Node.js 18+
    • npm
    • Port 3000 open

    Starting the Server¶

    cd src/server
npm install
npm start

    Server will output: 

    Server running at http://localhost:3000

    Connecting Clients¶

    1. Navigate to http://localhost:3000
    2. Enter your username
    3. Click "Continue"
    4. Start playing

    Multiple players can connect simultaneously.

    How Multiplayer Works¶

    Authentication¶

    Each player gets a unique token:

    Format: user_XXXXX_TIMESTAMP
Example: user_abc123_1673872800000

    Tokens are: - Generated on first visit - Stored in localStorage - Used for WebSocket authentication - Validated by server on each connection

    Real-Time Updates¶

    WebSocket Communication¶

    Server broadcasts player positions every frame:

    1. Client sends position update
    2. Server receives and validates
    3. Server broadcasts to all clients
    4. Clients update display

    Player Tracking¶

    The server tracks:

    • Username
    • Current position (x, y)
    • Last seen timestamp
    • Connection status

    Stale players are cleaned up after 5 minutes of inactivity.

    Admin Panel¶

    Accessing Admin Tools¶

    Navigate to http://localhost:3000/public/ to see:

    • Live player list
    • Player positions
    • Online/offline status
    • Server statistics

    Admin Features¶

    • View all connected players
    • Monitor real-time positions
    • Track player activity
    • Edit game settings
    • Manage maze configuration

    Technical Details¶

    WebSocket Events¶

    Client to Server¶

    {
  "type": "auth",
  "token": "user_abc123_1673872800000"
}

    {
  "type": "join",
  "username": "Player123"
}

    {
  "type": "move",
  "x": 5,
  "y": 10
}

    Server to Client¶

    {
  "type": "players",
  "players": [
    {"username": "Player1", "x": 1, "y": 1, "token": "user_..."},
    {"username": "Player2", "x": 5, "y": 10, "token": "user_..."}
  ],
  "timestamp": 1673872800000
}

    Server Architecture¶

    Express HTTP Server (Port 3000)
├── Leaderboard API
│   ├── GET /leaderboard
│   └── POST /leaderboard
├── Static Files
│   └── public/
└── WebSocket Server
    ├── Authentication
    ├── Player Management
    └── Broadcasting

    Data Flow¶

    Client 1
   │
   ├─── Move (x, y) ──┐
   │                  │
   │         WebSocket Server
   │                  │
   │    Broadcasts ───┤
   │    all players   │
   │                  │
Client 2 ◄────────────┘
   │
   └─── Receives Player1 position

    Performance Considerations¶

    Optimization Tips¶

    • Broadcast rate: ~10 updates/second
    • Player limit: Tested with 50+ concurrent players
    • Network: Requires stable internet connection
    • Latency: <100ms recommended

    Bandwidth Usage¶

    • Per player: ~1KB per second
    • Server upstream: Scales with player count

    Troubleshooting¶

    Connection Issues¶

    Problem: Cannot connect to multiplayer server

    Solution: 1. Verify server is running 2. Check port 3000 is accessible 3. Verify correct URL: http://localhost:3000 4. Check browser console for errors 5. Try restarting server

    Auth Errors¶

    Problem: "Invalid token" error

    Solution: 1. Clear localStorage: localStorage.clear() 2. Refresh browser 3. Server will generate new token

    Position Updates Delayed¶

    Problem: Other players' positions update slowly

    Solution: 1. Check network latency: ping localhost 2. Verify server is responsive 3. Check browser console for errors 4. Reduce player count if exceeding limits

    Security in Multiplayer¶

    Token Validation¶

    • Tokens format checked: user_*_*
    • Validated before processing moves
    • Invalid tokens rejected immediately

    Input Validation¶

    • Player positions validated (0 <= x,y <= maze size)
    • Usernames sanitized to 32 characters
    • Invalid moves rejected

    Data Protection¶

    • No sensitive data transmitted
    • All communication over WebSocket
    • No external auth servers
    • Session-based data only
