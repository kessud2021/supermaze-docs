    Maze Generation¶

    Learn how SuperMaze generates unique and solvable mazes.

    Algorithm¶

    SuperMaze uses the Recursive Backtracking algorithm (also known as Depth-First Search carving) to generate mazes.

    How It Works¶

    1. Start with a grid of all walls
    2. Pick a random cell as the current cell
    3. Mark current cell as passage
    4. While there are unvisited neighbors:
    5. Pick a random unvisited neighbor
    6. Remove the wall between current and neighbor
    7. Make neighbor the current cell
    8. Recursively repeat
    9. Backtrack when no unvisited neighbors exist

    Properties¶

    • Guaranteed Solvable: Always has at least one path to the goal
    • Unique Mazes: No two mazes are identical (random generation)
    • Perfect Mazes: No loops or isolated areas
    • Deterministic: Algorithm always completes successfully

    Maze Generation Process¶

    Step 1: Initialize Grid¶

    All cells start as walls (solid gray)
Size: 21x21 (configurable)

    Step 2: Carve Passages¶

    Recursively carve passages between cells
Creates corridors and open areas

    Step 3: Place Player and Goal¶

    Player: Random cell in upper-left area (red)
Goal: Random cell in lower-right area (green)
Guaranteed to be in open passage

    Step 4: Render Maze¶

    Display to player with current theme colors
Ready for gameplay

    Difficulty Levels¶

    Difficulty affects maze generation complexity:

    Level Size Complexity Avg. Time
    Easy Small Simple paths 1-2 min
    Medium 21x21 Balanced 3-5 min
    Hard Large Complex 6+ min

    Performance¶

    • Generation Time: <100ms for 21x21 maze
    • Memory: ~10KB per maze
    • Renderability: Instant (pre-computed)

    Algorithm Complexity¶

    • Time: O(N²) where N is grid size
    • Space: O(N²) for maze storage
    • Recursion Depth: O(N²) in worst case

    Variations¶

    SuperMaze uses the basic recursive backtracking algorithm. Other algorithms include:

    • Prim's Algorithm: Similar results, different approach
    • Kruskal's Algorithm: Another minimum spanning tree method
    • Hunt and Kill: Iterative alternative
    • Binary Tree: Faster but less balanced

    Guarantees¶

    Every generated maze has:

    • ✓ Exactly one solution (non-branching)
    • ✓ Player always reachable from goal
    • ✓ No unreachable areas
    • ✓ No loops or circles
    • ✓ Unique layout each time

    The maze generation makes every game unique and challenging!
